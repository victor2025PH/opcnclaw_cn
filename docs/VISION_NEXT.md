# 十三香小龙虾 — 下一步：从"能说话"到"真干活"

> 深度反思 | 2026-03-22
>
> 不是功能清单，是产品灵魂的思考

---

## 一、残酷的现实

我们做了很多——52个角色、117个技能、24个工具、407个测试。
但说实话，**Agent 目前只是换了个马甲的聊天机器人**。

```
用户说"帮我写营销方案"
→ 10 个 Agent 各自调用 AI 生成一段文字
→ CEO 把文字粘在一起
→ 输出一个 Markdown
→ 然后呢？

然后什么都没发生。
  没有人去执行这个方案。
  没有人去发朋友圈。
  没有人去写代码。
  没有人去做设计图。

这和直接问 ChatGPT 有什么区别？
区别只是多等了 2 分钟。
```

**这是我们最大的问题：Agent 会"说"，但不会"做"。**

---

## 二、什么叫"真干活"

### 现在的 Agent（只会说）

```
文案 Agent："建议标题用'一个人的AI公司'，文案如下..."
设计 Agent："建议用蓝紫渐变配色，Logo 可以考虑..."
程序员 Agent："建议用 FastAPI 框架，数据库用 PostgreSQL..."
运营 Agent："建议在小红书发布，第一周投放..."
```

全是**建议**。没有一个人真正动手。

### 应该的 Agent（真干活）

```
文案 Agent：
  → 直接生成 5 个版本的文案
  → 保存到 data/projects/marketing/文案_v1.md
  → 通知设计 Agent "文案好了，配图需要蓝紫渐变风格"

设计 Agent：
  → 根据文案生成 DALL-E/Midjourney 提示词
  → 调用 AI 绘图 API 生成 3 张候选图
  → 保存到 data/projects/marketing/海报_v1.png
  → 通知运营 Agent "设计稿好了"

程序员 Agent：
  → 在 data/projects/landing/ 目录创建文件
  → 写 index.html + style.css + script.js
  → 自动启动本地预览服务器
  → 通知 CEO "落地页已完成，预览地址 http://localhost:3000"

运营 Agent：
  → 组合文案+图片
  → 调用 publish_moment 发到朋友圈
  → 调用 send_wechat 发给 10 个种子用户
  → 生成投放计划表保存到 data/projects/marketing/投放计划.md
```

**每个 Agent 都产出实实在在的文件、执行实实在在的操作。**

## 四、技术路线（怎么实现）

### 第 1 阶段：Agent 产出文件（1 周）

```
核心改动：Agent execute() 时不只返回文字，还创建文件

1. 新建 src/server/project_workspace.py
   - create_project(name) → data/projects/xxx/
   - save_artifact(project, agent, filename, content)
   - list_artifacts(project)

2. 修改 agent_team.py
   - 每个 Agent 完成时自动保存到 project 目录
   - CEO 汇总报告保存为 README.md

3. 新增 API：
   - GET /api/projects — 项目列表
   - GET /api/projects/{id}/files — 文件列表
   - GET /api/projects/{id}/files/{name} — 下载文件

4. 前端：项目文件浏览器（替代纯聊天展示）
```

### 第 2 阶段：Agent 执行操作（1 周）

```
核心改动：Agent 不只生成文字，还执行操作

1. 文案 Agent：保存文件 + 通知设计
2. 设计 Agent：生成 AI 绘图提示词 + 调用绘图 API
3. 程序员 Agent：创建代码文件 + 运行检查
4. 运营 Agent：发微信 + 发朋友圈（使用现有工具）
5. 数据 Agent：生成 ECharts 图表 HTML
```

### 第 3 阶段：语音团队会议（1 周）

```
核心改动：团队结果用 TTS 播报，语音指挥修改

1. 每个 Agent 汇报时用 TTS 朗读（不同语速区分角色）
2. 用户语音指令识别 → 分发给对应 Agent
3. Agent 修改后再次 TTS 汇报
```

### 第 4 阶段：项目仪表盘 UI（1 周）

```
核心改动：新页面替代聊天界面

1. 新建 src/client/project.html — 项目仪表盘
2. 文件树 + 团队状态 + 进度条 + 对话流
3. 文件预览（Markdown 渲染 + 代码高亮）
4. 下载/分享/修改 操作
```
