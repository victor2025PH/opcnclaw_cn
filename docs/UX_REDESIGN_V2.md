# 交互重构方案 V2 — 让用户真正参与

---

## 一、当前问题

```
现在的流程（错误的）：
  用户点"写营销方案"
    → AI 直接部署团队
    → "已部署10人，请稍等"
    → 用户干等
    → 输出一堆不知道在说什么的内容

正确的流程：
  用户点"写营销方案"
    → AI 问"是什么产品的方案？"
    → 用户说"十三香AI"
    → AI 问"预算多少？目标用户？"
    → 用户说"5万，创业者"
    → AI 介绍团队每个人
    → 用户确认"开始"
    → 实时看到每个人在做什么
    → 可以随时跟任何一个人单独聊
    → 完成后审查、修改、确认
```

---

## 二、正确的 5 步交互流程

### 步骤 1：需求收集（对话式，不是一键启动）

```
用户点击"📣 写营销方案"

AI: "好的！请告诉我一些信息：
     1. 什么产品/项目？
     2. 目标用户是谁？
     3. 预算大概多少？
     4. 有什么特别要求吗？

     你可以一次性说，也可以一个一个回答。"

用户: "十三香AI工作队，面向创业者和小团队，
      预算5万，重点做线上推广"

AI: "收到！我来帮你组建团队 👇"
```

**关键**：AI 先问清楚需求，而不是直接执行。

### 步骤 2：团队亮相（介绍每个人）

```
AI: "📣 营销战队已就位！为你介绍团队成员：

     📡 CMO 陈总    — 负责整体营销策略和预算分配
     ✍️ 文案 小王    — 撰写产品文案和推广话术
     🎨 设计 小李    — 设计海报、Banner和视觉方案
     🔍 SEO 小张     — 搜索引擎优化和关键词布局
     📊 广告 小赵    — 信息流广告投放和ROI优化
     💬 社群 小陈    — 社群搭建和用户运营
     🚀 增长 小刘    — 裂变策略和用户增长
     📰 公关 小周    — 新闻稿和媒体关系
     🎪 品牌 小吴    — 品牌定位和视觉识别
     📈 数据 小郑    — 数据分析和效果追踪

     [🚀 开始执行]  [✏️ 调整需求]  [➖ 精简团队]"
```

**关键**：用户看到每个人，知道团队在做什么，有控制权。

### 步骤 3：实时执行（看到每个人在做什么）

```
     📡 CMO     ✅ 策略制定完成 (18s)          [查看] [对话]
     ✍️ 文案     🔄 正在撰写产品描述... 60%     [查看] [对话]
     🎨 设计     🔄 正在设计配色方案...          [查看] [对话]

     ...

     [⏸ 暂停]  [⏹ 停止]
```

**关键**：每个 Agent 后面有 [对话] 按钮。

### 步骤 4：单独与 Agent 沟通

## 三、桌宠造型重构

### 当前问题
- 是一个圆球 → 没有辨识度
- 有窗口边框 → 不像桌宠
- 不能自由拖动 → 体验差

### 目标效果

```
桌面上一只小龙虾（SVG/CSS 绘制）：

        🦞
       /||\
      / || \   ← 两只大钳子
     (  ◉◉  )  ← 圆眼睛会转
      \ -- /   ← 小嘴巴
       ||||    ← 身体（红色渐变）
      /    \   ← 小脚

状态动画：
  待命：轻微左右晃动 + 偶尔眨眼
  倾听：钳子张开 + 身体膨胀
  思考：眼睛转圈 + 头上冒问号
  指挥：钳子挥舞 + 小人飞出
  完成：开心跳动 + 挥钳子
  打盹：闭眼 + 身体缩小
```

### 技术实现

```
Tauri 窗口配置：
  - decorations: false（无边框）
  - transparent: true（透明背景）
  - always_on_top: true（置顶）
  - resizable: false
  - width: 120, height: 140
  - skip_taskbar: true（不在任务栏显示）

HTML/CSS：
  - 纯 CSS 绘制小龙虾（不用图片，确保清晰）
  - CSS animation 做各种状态动画
  - 背景 transparent

拖动：
  - Tauri: data-tauri-drag-region（已有）
  - 浏览器模式：mousedown+mousemove 模拟
```

## 四、核心代码修改点

### 修改 1：deploy_team 工具改为"收集需求"模式

```python
# 之前：直接部署团队
async def deploy_team(task, template):
    team = create_team(template, ai_call)
    asyncio.create_task(team.execute(task))
    return {"status": "executing"}

# 之后：先收集需求，用户确认后再执行
async def deploy_team(task, template):
    # 不立即执行，返回团队信息让用户确认
    tpl = get_template(template)
    agents = [AGENT_ROLES[r].to_dict() for r in tpl["roles"]]
    return {
        "status": "confirm_needed",
        "team_name": tpl["name"],
        "agents": agents,
        "task": task,
        "message": "以上是我为你组建的团队，确认开始吗？",
        "actions": ["开始执行", "调整需求", "换个团队"]
    }
```

### 修改 2：系统提示词改为"先问再做"

```
旧：用户让你做方案→立刻调用 deploy_team
新：用户让你做方案→先问清楚需求→介绍团队→等用户确认→才执行
```

### 修改 3：新增"与单个 Agent 对话" API

```
POST /api/agents/team/{team_id}/agent/{agent_id}/chat
{
  "message": "标题不够吸引人，能改吗？"
}
→ 该 Agent 单独回答，不影响其他人
```

### 修改 4：pet.html 改为小龙虾 SVG

```html
<!-- 替换圆球为小龙虾 SVG -->
<svg class="pet-lobster" viewBox="0 0 120 140">
  <!-- 钳子 -->
  <!-- 身体（红色渐变） -->
  <!-- 眼睛（会转动） -->
  <!-- 脚 -->
</svg>
```

### 修改 5：Tauri 桌宠窗口去边框

