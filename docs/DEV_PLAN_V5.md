## 二、模块一：手机远程控制电脑（AnyDesk 级）

### 2.1 现有基础

已有能力（v4.4 已实现）：
- ✅ `DesktopStreamer`：屏幕截图 + JPEG 压缩 + Base64 传输
- ✅ `handle_command()`：鼠标移动/点击/双击/拖拽/滚轮/按键/快捷键/输入
- ✅ WebSocket `/ws` 双向通信
- ✅ HTTPS 证书自动生成（手机浏览器可访问）
- ✅ OCR 屏幕文字识别

缺失能力：
- ❌ 手机端触控→鼠标映射
- ❌ 自适应画质（带宽感知）
- ❌ 虚拟键盘
- ❌ 剪贴板同步
- ❌ 多指手势支持

### 2.2 技术方案

#### 传输协议选择

| 方案 | 延迟 | 复杂度 | 选择 |
|------|------|--------|------|
| WebSocket JPEG 帧 | 100-300ms | 低 | ✅ **首选**（已有基础） |
| WebRTC | 50-100ms | 高 | 后续升级 |
| VNC 协议 | 200-500ms | 中 | 不选（需额外软件） |

**决策**：先用 WebSocket JPEG 帧（已有 90% 代码），后续升级 WebRTC。

#### 画质自适应

```python
# 根据网络带宽自动调整
class AdaptiveQuality:
    PROFILES = {
        "high":   {"fps": 15, "quality": 60, "scale": 1.0},    # WiFi
        "medium": {"fps": 10, "quality": 40, "scale": 0.75},   # 4G
        "low":    {"fps": 5,  "quality": 25, "scale": 0.5},    # 3G
    }

    def adapt(self, rtt_ms, bandwidth_kbps):
        if bandwidth_kbps > 5000:  return "high"
        if bandwidth_kbps > 1000:  return "medium"
        return "low"
```

## 三、模块二：文件双向传输（手机 ↔ 电脑）

### 3.1 使用场景

```
场景 A: 手机 → 电脑
  手机拍照 → 拖入十三香 → 电脑桌面出现照片
  手机选文件 → 上传 → 电脑指定目录

场景 B: 电脑 → 手机
  电脑选文件 → 推送 → 手机通知 → 点击下载
  AI 截图结果 → 自动推送到手机相册

场景 C: AI 辅助传输
  "把桌面的报告发到我手机" → AI 找到文件 → 推送
  "把手机上的照片放到电脑桌面" → AI 接收 → 保存
```

### 3.2 技术方案

```
手机端                    服务器                    电脑端
  │                        │                        │
  │  POST /api/files/upload│                        │
  │───────────────────────>│                        │
  │  (multipart/form-data) │  保存到 data/transfers/│
  │                        │────────────────────────>│
  │                        │  EventBus: file_received│
  │                        │                        │
  │                        │  POST /api/files/push   │
  │                        │<────────────────────────│
  │  EventBus: file_pushed │                        │
  │<───────────────────────│                        │
  │  GET /api/files/{id}   │                        │
  │───────────────────────>│  下载文件               │
```

### 3.3 API 设计

```
POST   /api/files/upload          # 手机上传文件到电脑
GET    /api/files/list            # 文件列表（传输历史）
GET    /api/files/{id}/download   # 下载文件
POST   /api/files/push            # 电脑推送文件到手机
DELETE /api/files/{id}            # 删除传输文件
GET    /api/files/status          # 传输状态（进度/速度）

# AI 辅助
POST   /api/files/ai-send        # "把桌面的xx文件发到手机"
```

| `src/server/routers/files.py` | 文件传输 API 路由 | 1天 |

### 3.5 技术细节

**大文件分片上传**：
```
文件 > 10MB → 自动分片（每片 2MB）
每片独立上传 → 服务器合并
断点续传：记录已上传片段，断网后续传
```

**安全**：
```
传输文件 24h 后自动清理
单文件上限 500MB
总空间上限 2GB
PIN 码保护（与现有 PIN 系统集成）
```

## 四、模块三：一键部署 Agent 团队（核心创新）

### 4.1 设计理念

```
传统方式: 用户 ←→ 1个AI（所有问题问同一个AI）

十三香方式: 用户 ←→ Agent 团队（每个Agent专精一个领域）

  用户: "帮我做一个新产品发布方案"
    │
    ▼
  [CEO Agent] 接收任务 → 拆解为子任务
    │
    ├→ [研究员] 调研市场数据、竞品分析
    ├→ [分析师] 分析用户画像、定价策略
    ├→ [写手] 撰写产品文案、新闻稿
    ├→ [设计师] 描述视觉方案（让AI生成图片prompt）
    └→ [运营] 制定推广计划、渠道分配
         │
         ▼
    [CEO Agent] 汇总审核 → 输出完整方案
```

### 4.4 团队工作流引擎

```python
class AgentTeam:
    """一键部署的 Agent 团队"""

    def __init__(self, template: str):
        self.agents: Dict[str, Agent] = {}
        self.task_board: List[Task] = []
        self.shared_context: Dict = {}
        self.message_bus: List[Message] = []

    async def execute(self, user_request: str):
        """
        1. CEO 接收用户需求
        2. CEO 拆解为子任务，分配给各 Agent
        3. 各 Agent 并行执行（有依赖的按序）
        4. Agent 间可以互相提问/审核
        5. CEO 汇总所有结果
        6. 输出最终方案
        """
```

## 八、技术风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| Agent 间通信死锁 | 团队任务卡死 | 设置 5 分钟超时 + 依赖图检测 |
| 远程桌面延迟高 | 操作体验差 | 帧差分 + 自适应画质 + 本地预测 |

| 手机浏览器兼容性 | 触控失灵 | 渐进增强 + 多浏览器测试 |
