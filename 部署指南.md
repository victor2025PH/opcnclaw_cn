# OpenClaw Voice — 部署指南

## 部署方案对比

| 方案 | 适用场景 | 难度 | 手机访问范围 |
|------|----------|------|------------|
| **A. 本机服务（推荐）** | 家/公司固定电脑 | ⭐ | 同一 WiFi |
| **B. 内网穿透** | 出门也要用 | ⭐⭐ | 全球任意网络 |
| **C. 云服务器** | 团队共用 | ⭐⭐⭐ | 全球任意网络 |

## 方案 A：本机作为永久服务器（最推荐）

### 一键安装（开机自启）

```powershell
# 以管理员身份打开 PowerShell，执行：
cd D:\xlx2026\openclaw-voice
.\install-service.ps1
```

安装后：
- ✅ 开机自动启动，无需手动运行
- ✅ 崩溃自动重启
- ✅ 日志保存到 `logs/` 目录
- 📱 手机扫码地址：`http://电脑IP:8766/qr`
- 💻 完整版地址：`https://电脑IP:8765/app`

### 手动管理服务

```powershell
Start-Service OpenClawVoice      # 启动
Stop-Service  OpenClawVoice      # 停止
Restart-Service OpenClawVoice   # 重启
Get-Service   OpenClawVoice      # 查看状态

# 卸载服务：
.\uninstall-service.ps1
```

### 查看实时日志

```powershell
Get-Content logs\service-stdout.log -Wait -Tail 50
```

## 方案 B：内网穿透（随时随地访问）

让外网手机也能访问你电脑上的 OpenClaw，不限于同一 WiFi。

### 快速体验（临时地址，免注册）

```powershell
.\setup-tunnel.ps1
# 选择 1（临时隧道）
# 会显示类似 https://xxxx.trycloudflare.com 的地址
# 手机扫码即可，但地址每次重启会变
```

### 固定地址（需要自己的域名）

1. 注册 [Cloudflare 免费账号](https://dash.cloudflare.com)
2. 把你的域名托管到 Cloudflare
3. 运行 `.\setup-tunnel.ps1`，选择 2

配置完成后，手机在任何网络都能访问 `https://ai.你的域名.com/chat`

## 方案 C：云服务器部署

适合需要多人使用、或者本机需要关机的情况。

### 最低配置要求
- CPU：2 核（STT 用 Whisper base 够用）
- 内存：4 GB
- 系统：Ubuntu 22.04
- 推荐：阿里云/腾讯云轻量服务器（约 ¥40/月）

### 部署步骤

```bash
# 1. 服务器上安装 Python 3.11+
sudo apt update && sudo apt install -y python3.11 python3.11-venv git ffmpeg

# 2. 上传项目（或 git clone）
scp -r D:\xlx2026\openclaw-voice user@服务器IP:/opt/openclaw

# 3. 安装依赖
cd /opt/openclaw
python3.11 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 4. 配置环境变量
cp .env.example .env
nano .env   # 填入 API Key 等配置

# 5. 申请正式 SSL 证书（免费）
sudo apt install certbot
sudo certbot certonly --standalone -d ai.你的域名.com
# 证书路径：/etc/letsencrypt/live/ai.你的域名.com/

# 6. 修改 .env 使用正式证书
# OPENCLAW_CERT_PATH=/etc/letsencrypt/live/ai.你的域名.com/fullchain.pem
# OPENCLAW_KEY_PATH=/etc/letsencrypt/live/ai.你的域名.com/privkey.pem

# 7. 配置 systemd 服务（开机自启）
sudo tee /etc/systemd/system/openclaw.service > /dev/null << 'EOF'
[Unit]
Description=OpenClaw Voice AI Server
After=network.target

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/opt/openclaw
ExecStart=/opt/openclaw/venv/bin/python -m src.server.main
Restart=always
RestartSec=5
Environment=PYTHONUNBUFFERED=1

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl enable openclaw
sudo systemctl start openclaw
sudo systemctl status openclaw
```

### 云端注意事项
- 远程桌面控制功能在云服务器上不可用（需要本地 `pyautogui`）
- STT Whisper 在无 GPU 的云服务器上很慢，建议改用 Zhipu 或 OpenAI 的 STT API
- 摄像头/手势功能仍在用户手机端运行，不受影响

## 常见问题

**Q: 手机连不上，提示网络错误**
- 确认手机和电脑在同一 WiFi
- 检查防火墙：`netsh advfirewall firewall show rule name="OpenClaw*"`
- 检查 IP：扫码页面显示的 IP 是否是 `192.168.x.x`

**Q: HTTPS 证书警告，iPhone 用不了摄像头**
- 访问 `https://电脑IP:8765/setup` 安装自签证书
- iPhone 需要去 设置→通用→关于本机→证书信任设置 中开启信任

**Q: 服务器重启后模型重新加载很慢**
- 正常现象，Whisper 首次加载需要 30-60 秒
- 查看日志确认 `OpenClaw Voice server ready!` 出现后再使用

**Q: 修改了代码，如何热更新**
```powershell
Restart-Service OpenClawVoice
```
